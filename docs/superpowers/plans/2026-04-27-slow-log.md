# Slow Log Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a per-transaction slow log to pg_doorman that surfaces pool-side latency contributors invisible to PostgreSQL's `log_min_duration_statement`.

**Architecture:** Per-`Client` accumulator (`SlowLogTracker`) collects wait/idle/xact timings + byte deltas during each transaction. On xact completion, three thresholds (configured under `[slow_log]`) determine whether one `warn!()` line is emitted in logfmt. Hot path adds at most one new `now()` call when enabled; zero overhead when all thresholds are 0. Three runtime atomics back the thresholds for SIGHUP-friendly reload.

**Tech Stack:** Rust, tokio, `quanta::Instant`, `log` crate, `serde_derive` (TOML/YAML config), existing pg_doorman BDD framework (`cucumber` crate, `tests/bdd/`).

**Spec:** `docs/superpowers/specs/2026-04-27-slow-log-design.md`

**Pre-commit policy:** Every Rust-touching commit goes through the user's mandatory pre-commit code review agent (see `~/.claude/CLAUDE.md` "Pre-commit code review (Rust)"). Each commit also runs `cargo fmt` and `cargo clippy -- -D warnings` locally.

//! Per-transaction slow log accumulator and emit logic.
//!
//! See `docs/superpowers/specs/2026-04-27-slow-log-design.md` for design.

use std::sync::atomic::{AtomicBool, AtomicU64, Ordering};

pub(crate) static SLOW_LOG_WAIT_US: AtomicU64 = AtomicU64::new(0);
pub(crate) static SLOW_LOG_IDLE_US: AtomicU64 = AtomicU64::new(0);
pub(crate) static SLOW_LOG_XACT_US: AtomicU64 = AtomicU64::new(0);
pub(crate) static SLOW_LOG_ENABLED: AtomicBool = AtomicBool::new(false);

pub(crate) const HINT_WAIT_QUEUE: &str = "pool exhausted, all backends busy";
pub(crate) const HINT_IDLE_IN_TX: &str = "client held tx idle between statements";
pub(crate) const HINT_LONG_XACT:  &str = "long active transaction";

/// Apply config thresholds to the runtime atomics. Called once on startup
/// and on every config reload (SIGHUP / admin RELOAD).
pub fn apply_config(cfg: &crate::config::SlowLogConfig) {
    SLOW_LOG_WAIT_US.store(cfg.wait_threshold_ms.saturating_mul(1_000), Ordering::Relaxed);
    SLOW_LOG_IDLE_US.store(cfg.idle_threshold_ms.saturating_mul(1_000), Ordering::Relaxed);
    SLOW_LOG_XACT_US.store(cfg.xact_threshold_ms.saturating_mul(1_000), Ordering::Relaxed);
    let any_set = cfg.wait_threshold_ms | cfg.idle_threshold_ms | cfg.xact_threshold_ms > 0;
    SLOW_LOG_ENABLED.store(any_set, Ordering::Relaxed);
}

/// Borrowed contextual fields used when emitting a slow xact line.
#[derive(Copy, Clone)]
pub(crate) struct SlowLogCtx<'a> {
    pub user: &'a str,
    pub db: &'a str,
    pub client_id: u64,
    pub server_pid: i32,
    pub mode_str: &'static str, // "transaction" | "session"
    pub xact_id: u64,
}

#[derive(Default)]
pub(crate) struct SlowLogTracker {
    pub(crate) enabled: bool,
    pub(crate) queries: u32,
    pub(crate) max_wait_us: u64,
    pub(crate) idle_us: u64,
    pub(crate) bytes_c2s_at_begin: u64,
    pub(crate) bytes_s2c_at_begin: u64,
    pub(crate) last_query_end: Option<quanta::Instant>,
}

impl SlowLogTracker {
    /// Reset all per-xact state.
    fn reset(&mut self) {
        self.enabled = false;
        self.queries = 0;
        self.max_wait_us = 0;
        self.idle_us = 0;
        self.bytes_c2s_at_begin = 0;
        self.bytes_s2c_at_begin = 0;
        self.last_query_end = None;
    }
}

    /// Snapshot the global `SLOW_LOG_ENABLED` flag and store baseline byte
    /// counters. Called at the start of every transaction.
    #[inline]
    pub fn begin_xact(&mut self, bytes_c2s_baseline: u64, bytes_s2c_baseline: u64) {
        self.queries = 0;
        self.max_wait_us = 0;
        self.idle_us = 0;
        self.bytes_c2s_at_begin = bytes_c2s_baseline;
        self.bytes_s2c_at_begin = bytes_s2c_baseline;
        self.last_query_end = None;
        self.enabled = SLOW_LOG_ENABLED.load(Ordering::Relaxed);
    }

    #[inline]
    pub fn observe_wait(&mut self, wait_us: u64) {
        if !self.enabled {
            return;
        }
        if wait_us > self.max_wait_us {
            self.max_wait_us = wait_us;
        }
    }

    #[inline]
    pub fn observe_query_end(&mut self, now: quanta::Instant) {
        if !self.enabled {
            return;
        }
        self.queries = self.queries.saturating_add(1);
        self.last_query_end = Some(now);
    }

    #[inline]
    pub fn observe_message_start(&mut self, now: quanta::Instant) {
        if !self.enabled {
            return;
        }
        if let Some(last) = self.last_query_end.take() {
            // Saturating: clock can't go backwards under quanta, but be
            // defensive against pathological CPU migration on unsupported
            // platforms.
            let dt = now.saturating_duration_since(last).as_micros() as u64;
            self.idle_us = self.idle_us.saturating_add(dt);
        }
    }
```

    /// Evaluate thresholds and emit a single `warn!()` line if any threshold
    /// was tripped. Always resets the tracker before returning.
    #[inline]
    pub fn maybe_emit(
        &mut self,
        ctx: SlowLogCtx<'_>,
        xact_us: u64,
        bytes_c2s_now: u64,
        bytes_s2c_now: u64,
    ) {
        if !self.enabled {
            self.reset();
            return;
        }

        let wait_us = self.max_wait_us;
        let idle_us = self.idle_us;

        let wait_t = SLOW_LOG_WAIT_US.load(Ordering::Relaxed);
        let idle_t = SLOW_LOG_IDLE_US.load(Ordering::Relaxed);
        let xact_t = SLOW_LOG_XACT_US.load(Ordering::Relaxed);

        let wait_hit = wait_t > 0 && wait_us >= wait_t;
        let idle_hit = idle_t > 0 && idle_us >= idle_t;
        let xact_hit = xact_t > 0 && xact_us >= xact_t;

        if !(wait_hit || idle_hit || xact_hit) {
            self.reset();
            return;
        }

        let exec_us = xact_us.saturating_sub(wait_us).saturating_sub(idle_us);

        let (reason, hint) = if wait_us >= idle_us && wait_us >= exec_us {
            ("WAIT_QUEUE", HINT_WAIT_QUEUE)
        } else if idle_us > exec_us {
            ("IDLE_IN_TX", HINT_IDLE_IN_TX)
        } else {
            ("LONG_XACT", HINT_LONG_XACT)
        };

        let bytes_c2s = bytes_c2s_now.saturating_sub(self.bytes_c2s_at_begin);
        let bytes_s2c = bytes_s2c_now.saturating_sub(self.bytes_s2c_at_begin);

        log::warn!(
            "[{}@{} #c{}] slow xact reason={} hint=\"{}\" pid={} mode={} xact_id={} \
             queries={} wait={}ms xact={}ms idle={}ms bytes={}/{}",
            ctx.user, ctx.db, ctx.client_id, reason, hint, ctx.server_pid, ctx.mode_str,
            ctx.xact_id, self.queries, wait_us / 1_000, xact_us / 1_000, idle_us / 1_000,
            bytes_c2s, bytes_s2c,
        );

        self.reset();
    }
```

## Self-Review Notes (for the implementing agent)

- The spec calls out **approximate** bytes (per-pool counters loaded into per-xact baselines). Tests in Task 4 deliberately use trivial values; live values will be polluted by other clients in the same pool — that's by design.
- All threshold comparisons use `>=`, not `>`. A wait of exactly `wait_threshold_ms × 1000` µs counts as a hit.
- `--test-threads=1` is required for the slow_log unit-test module because it mutates process-global atomics (`SLOW_LOG_*_US`) and the global logger. Other modules in the project may or may not need it — keep `cargo test --lib` invocations on a single thread for slow-log work.
- Hot path overhead when `enabled=false`: one atomic load + one branch per `observe_*` call. When *all thresholds = 0*, the global flag stays `false` and `begin_xact` snapshots that — no further atomic loads happen until the next `begin_xact`.
- The `xact_id` is per-`Client` (per session). It does not collide with anything; it's purely a grep-friendly key.
- If `cargo clippy` complains about `bytes_c2s_at_begin`/`bytes_s2c_at_begin` being unused after the saturating subtractions but no actual emit happens (cold paths), `#[allow(dead_code)]` is **not** the answer — those fields are read in `maybe_emit`, so clippy will not flag them.
