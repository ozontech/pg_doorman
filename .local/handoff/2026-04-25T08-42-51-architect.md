## Context

Branch `feature/patroni-failover-discovery`. Two issues from the audit, scoped by the user:

1. **`nostream` tag not filtered** in `select_candidates`. Decision: keep filter as-is. `nostream` means cascade replica (data via another replica, not direct streaming from leader) — it's a live, valid read target. Filtering would shrink the pool with no safety benefit. Action: document the deliberate choice + add a unit test so nobody "fixes" it later.

2. **Failed inflight future poisons the next 1s window.** `fetch_cluster_coalesced` keeps a `Shared` future in `inflight` for `INFLIGHT_STALENESS = 1s`. If the request fails, every coalesced caller in that window gets the same cached error — even after Patroni recovers. Action: clear the slot when the creator's await resolved to `Err`.

Edge case "all members filtered out" stays correct as-is: empty candidates → `Err("no eligible members in /cluster response")` → client error → primary stays blacklisted → next attempt re-discovers. No relaxed-retry needed. This is the reason Phase 1 must NOT add another filter.

`futures = "0.3"` in Cargo.toml.

### Phase 1 — document `nostream`

In `src/pool/failover.rs`, replace the doc comment on `fn select_candidates` with:

```rust
/// Filter and sort members from a /cluster response.
///
/// Excluded:
/// - non-running members (state ≠ "streaming"/"running")
/// - `noloadbalance`: admin-disabled for query traffic
/// - `nofailover`: admin-disabled for promotion (writes would fail)
/// - `archive`: passive archive replica
///
/// Not excluded:
/// - `nostream`: cascade replicas reach data via another replica.
///   They are valid read targets — possibly higher lag, but live.
///
/// Sorted: sync_standby first, then replica, then others (including leader).
```

Add a unit test in the existing `mod tests`:

```rust
#[test]
fn select_candidates_keeps_nostream() {
    let mut cascade = make_member("pg-cascade", Role::Replica, "streaming", "10.0.0.1", 5432);
    cascade.tags.nostream = true;
    let normal = make_member("pg-normal", Role::Replica, "streaming", "10.0.0.2", 5432);
    let candidates = select_candidates(&[cascade, normal]);
    assert_eq!(candidates.len(), 2);
}
```

### Phase 2 — clear failed inflight

In `FailoverState::fetch_cluster_coalesced`, after the existing duration-record block, add:

```rust
// Clear cached future on error so the next caller retries fresh
// instead of inheriting the same failure for up to INFLIGHT_STALENESS.
// Only the creator clears: joiners would race with whoever installed
// the next inflight.
if is_creator && result.is_err() {
    let mut guard = self.inflight.lock().await;
    *guard = None;
}
```

Remove the existing `// No clearing of inflight here — staleness-based expiry handles it. Clearing here would race...` comment.

Add a unit test:

```rust
#[tokio::test]
async fn failed_inflight_does_not_poison_next_caller() {
    let state = FailoverState::new(
        "test_pool_inflight_fail".to_string(),
        vec!["http://127.0.0.1:1/cluster".to_string()],
        Duration::from_secs(10),
        Duration::from_millis(100),
        Duration::from_millis(200),
        30_000,
    ).unwrap();

    let before = crate::prometheus::FAILOVER_DISCOVERY_TOTAL
        .with_label_values(&["test_pool_inflight_fail"]).get();
    let _ = state.fetch_cluster_coalesced().await;
    let _ = state.fetch_cluster_coalesced().await;
    let after = crate::prometheus::FAILOVER_DISCOVERY_TOTAL
        .with_label_values(&["test_pool_inflight_fail"]).get();

    assert_eq!(after - before, 2);
}
```

If `fetch_cluster_coalesced` is currently private — make it `pub(super)` or `pub(crate)`. Pool name in the test is unique so the global metrics REGISTRY doesn't collide with other tests.
