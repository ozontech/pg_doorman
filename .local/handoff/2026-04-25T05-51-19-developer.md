## Changes

### Phase 1 — `select_candidates` doc comment + `nostream` test

Replaced the one-liner doc comment on `fn select_candidates` with the
explicit Excluded / Not excluded breakdown from the plan. The `nostream`
bullet documents why cascade replicas remain valid read candidates and
deters future "fixes" that would filter them out.

Added unit test `select_candidates_keeps_nostream` that builds two replicas
(one with `nostream = true`, one without) and asserts both pass the filter.

### Phase 2 — clear failed inflight + coalescing-recovery test

Removed the prior comment that explicitly opted out of clearing the inflight
slot on error. Replaced it with a creator-only clear-on-error block:

```rust
if is_creator && result.is_err() {
    let mut guard = self.inflight.lock().await;
    *guard = None;
}
```

Bumped `fetch_cluster_coalesced` from private to `pub(super)` so the unit
test in the same file's `mod tests` can call it directly. (Plan permitted
this; `pub(crate)` not needed.)

Added `#[tokio::test] failed_inflight_does_not_poison_next_caller` that
points discovery at `http://127.0.0.1:1/cluster` (guaranteed connection
refused), then calls `fetch_cluster_coalesced` twice sequentially. Both
calls fail. The test reads the `FAILOVER_DISCOVERY_TOTAL` counter before
and after and asserts the delta is 2 — proving the second caller created
a fresh inflight rather than reusing the failed one. Pool name
`test_pool_inflight_fail` is unique to avoid metric label collisions with
other tests.
